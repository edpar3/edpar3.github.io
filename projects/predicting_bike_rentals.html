Predicting Bike Rentals
This project takes data from a bike rental system and uses three machine learning models to predict the number of rentals per hour. The best results appear in the random forest model, but that model is likely overfit. More analysis would be necessary to select the best model.

def assign_label(hour):
  if hour < 6:
    label = 4
  elif hour < 12:
    label = 1
  elif hour < 18:
    label = 2
  elif hour < 24:
    label = 3
  return label

# Train and test a linear regression model.

from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error
import numpy as np

lnr = LinearRegression()
lnr.fit(train[drivers], train.cnt)
predictions = lnr.predict(test[drivers])
mse = mean_squared_error(predictions, test.cnt)
rmse = np.sqrt(mse)
print(rmse)

# Train and test a random forest model.

from sklearn.ensemble import RandomForestRegressor

    max_depth=None, 
    min_samples_leaf=4,
    random_state=1
    )

rfreg.fit(train[drivers], train.cnt)

print(rmse)

Random Forest Results
We get better results still with random forest. However, the best RMSE results will likely arise from overfitting.

Some Potential Next Steps
(1) Train the models using cross validation to prevent overfitting.
(2) Calculate additional features, such as:
-- An index combining temperature, humidity, and wind speed.
-- Predict casual and registered instead of cnt.
